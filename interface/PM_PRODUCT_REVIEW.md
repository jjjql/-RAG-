# 北向接口 — 产品对齐清单（@产品经理）

> 架构侧事实来源：`interface/openapi.yaml`。  
> **本轮**：@PM 与 @Architect 已就下列条目**书面对齐**；**开发实现以 OpenAPI 落盘字段为准**。当前契约 **`info.version: 0.2.3`** 已纳入 §0 要点（**0.2.3**：管理端网关侧鉴权裁剪，见 §0）。

---

## 0. 产品与架构对齐纪要（2026-03-26）

| 维度 | 结论 |
|---|---|
| **用户 SSE 鉴权** | 本期 **无** 用户端 API Key / mTLS / Bearer；依赖部署网络边界（与 PRD §1.3 一致）。 |
| **精确查询维度** | 用户侧精确规则命中按 **`scope` + `key`** 两维，与管理员精确规则一致；`scope` 典型为**一级产品线 ID**；`query` 仍服务正则 / 语义 / RAG（见 `QARequest`）。 |
| **错误码** | `ErrorBody.code` 为**稳定中文短语文本**（可分支）；`message` 中文为主。 |
| **管理端鉴权** | **本期网关不校验**凭证；与 **用户 SSE** 一致，由**部署网络边界 / 前置网关**承担访问控制；若未来在网关内实现 JWT/API Key，须递增 OpenAPI 主版本并更新 PRD。 |
| **配置/产品版本可见性** | **不使用** 响应头 `X-Config-Version`；**产品版本**在 SSE **`event:meta` / `event:done`** 的 **`productVersion`**（建议 meta 首条）。 |
| **UDS** | Go↔Python **内部契约**（`specs/architecture/contracts/`），**不在本清单逐条评审**。 |

## 1. 用户问答 `POST /v1/qa`（**SSE 流式**）

| # | 议题 | 架构当前假设 | 请产品确认（**已与架构对齐**） |
|---:|---|---|---|
| 1.1 | **用户鉴权** | **架构已锁定（本期）**：北向 **SSE** **不使用 API Key / mTLS / Bearer**；依赖部署网络边界。 | **已锁定**（同 PRD §1.3） |
| 1.2 | **作用域 `scope` + 精确键 `key`** | 精确规则按 **`scope` + `key`** 与管理端一致；`scope` 典型为一级产品线 ID；未传 `key` 则不做精确键命中。 | **已对齐** |
| 1.3 | **会话 `sessionId`** | 可选，透传/记录，**不参与**规则 KEY 匹配。 | **已对齐**：**可选即可** |
| 1.4 | **应答来源 `source`** | `event:done`：`rule_exact` / `rule_regex` / `semantic_cache` / `rag`。 | **已对齐**：与对外话术一致；**「智能问答」仅对应 `rag`**；`semantic_cache` 为语义缓存命中 |
| 1.5 | **答案形态** | `delta.text` 顺序拼接；DAT 可为 JSON 文本由业务解析。 | **已对齐**：**本期仅纯文本增量**；**不要** citations 等额外 SSE 事件（下版另立项） |
| 1.6 | **错误与前置失败** | HTTP 4xx/5xx + JSON；流内可 `event:error`。 | **已对齐**：**接受** |
| 1.7 | **Accept 与 406** | 契约要求 `Accept: text/event-stream`。 | **已对齐**：**须返回 406**（缺省或非流 Accept 时） |
| 1.8 | **非流式兼容** | 当前仅 SSE。 | **已对齐**：**仅 SSE**；**不要** JSON 双形态（本期） |
| 1.9 | **心跳与超时** | SSE `:` 注释心跳；空闲超时由实现定义。 | **已对齐**：**本期不在 PRD 写死总时长上限**；由实现与运维配置 |
| 1.10 | **错误码表** | `code` 形态与是否另附码表。 | **已对齐**：**中文稳定 `code`**；**不强制**单独印刷码表，可在运维 wiki 维护速查 |
