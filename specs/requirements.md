# 产品需求规格说明（PRD）- 智能问答网关（对外能力）

> 版本：v0.2（产品视角，不含实现与内部架构）
>
> 读者：产品/业务/运营；本文不描述系统内部模块、技术选型与部署方式。

## 1. 背景与目标

### 1.1 产品要解决什么问题

- 用户提出问题后，系统应尽可能**快速**给出回答。
- 对于可标准化、可预判的问题，系统管理员可预先配置**精确匹配**与**正则匹配**规则，使系统在用户无感的情况下实现**极速应答**。
- 对于无法被规则覆盖的问题，系统应能继续走**智能问答**（由外部知识/模型能力提供最终答案）；本文仅约束**本网关产品对外承诺的体验与性能边界**，不约束外部服务的实现细节。

### 1.2 角色定义（仅两类）

| 角色 | 说明 |
|---|---|
| **用户** | 发起提问并获得回答的一方（不关心系统内部如何工作）。 |
| **系统管理员** | 维护“规则应答库”（精确 KEY / 正则表达式）及其对应应答内容（DAT）的一方。 |

### 1.3 不在本期产品范围（对外声明）

- 外部知识库建设、模型训练、RAG 流水线细节与效果调优（由外部服务/供应商负责）。
- 计费、复杂多租户商业化策略（可作为后续版本）。
- **用户北向问答（含 SSE）本期不要求 API Key / mTLS**：由部署与网络边界保障可达性与滥用防护；若未来版本需要用户端鉴权，单独立项并更新对外契约。

## 2. 名词解释（业务语言）

| 名词 | 含义 |
|---|---|
| **精确 KEY** | 管理员配置的“完全相等匹配”的关键字/关键短语（用于命中后直接返回固定应答）。 |
| **正则表达式规则** | 管理员配置的模式匹配规则；用户问题满足该模式时，返回对应应答。 |
| **DAT** | 命中规则后返回给用户的应答内容（可为纯文本，也可为结构化 JSON 文本，由业务约定）。 |
| **作用域（可选）** | 用于把规则分区隔离的维度；与契约对齐时典型示例为**公司一级产品线 ID**（亦可与部门/环境等组合约定）。用户侧**精确命中**须与 **`key` 同时**提供，与管理员配置的 `(scope, key)` 一致。 |
| **P99** | 99% 的请求体验不劣于该延迟指标（用于描述“绝大多数用户”的体感）。 |

## 3. 功能需求表（FR）

> 验收口径使用**用户可理解的结果**描述；不涉及内部实现步骤。

### 3.1 系统管理员：规则与应答管理

| 编号 | 需求名称 | 需求描述 | 输入 | 输出 | 业务规则 | 验收标准（产品） |
|---|---|---|---|---|---|---|
| FR-A01 | 精确 KEY / DAT 管理 | 管理员可维护“精确匹配规则库” | KEY、DAT；可选作用域 | 成功/失败；列表/详情 | 同一作用域下 KEY 不可重复；DAT 不能为空（除非业务允许空应答），精确优先于正则；| 增删改查均可用；重复 KEY 被拒绝并给出明确原因 |
| FR-A02 | 正则表达式 / DAT 管理 | 管理员可维护“模式匹配规则库” | 正则表达式、DAT；可选作用域 | 成功/失败；列表/详情 | 非法正则表达式必须被拒绝；规则冲突时必须有明确优先级策略（默认策略需在需求确认稿中写死一条），正则按 priority 降序；同 priority 按更新时间最新优先  | 非法表达式无法保存；合法表达式可保存；命中结果符合既定优先级 |
| FR-A03 | 规则生效时效 | 管理员变更规则后，应对线上问答行为产生可预期影响 | 变更操作 | 变更结果 | 变更应在1秒内对用户提问生效；不要求用户重启客户端 | 变更后，在约定时间内用相同问题验证应答已按新规则变化 |


### 3.2 用户：提问与获得回答

| 编号 | 需求名称 | 需求描述 | 输入 | 输出 | 业务规则 | 验收标准（产品） |
|---|---|---|---|---|---|---|
| FR-U01 | 提问 | 用户提交自然语言问题 | 用户问题文本；可选上下文字段（由业务定义，如会话 ID） | 回答内容；可选解释字段（若业务需要） | 必须返回可理解的错误信息（网络/服务不可用/无权限等） | 正常场景必有回答；异常场景错误信息清晰 |
| FR-U02 | 规则极速应答 | 用户问题命中精确 KEY 或正则规则时，返回对应 DAT | 用户问题 | DAT | 命中规则时，应优先返回规则应答（相对“智能问答”路径更快） | 用一组“已配置规则”的固定问题验收：返回内容与 DAT 一致 |
| FR-U03 | 智能问答应答 | 用户问题未命中规则时，返回由外部知识/模型能力生成的答案 | 用户问题 | 答案 | 答案来源对外可标注为“智能问答”（可选，由产品决定） | 用一组“未配置规则”的问题验收：仍能得到合理答案（由业务定义“合理”） |

## 4. 性能需求表（NFR）

> 说明：以下指标描述**用户侧体感**相关的延迟承诺；不把外部知识/模型服务的计算时间算入“本网关承诺范围”（但端到端总耗时仍可能受外部服务影响，需在产品说明中区分）。

| 编号 | 场景（产品描述） | 指标 | 承诺边界（产品口径） | 验收方法（产品可执行） |
|---|---|---|---|---|
| NFR-P01 | **规则极速应答**：命中精确 KEY 或正则规则并返回 DAT | **P99 < 15ms** | 从用户请求进入到返回 DAT 的链路中，**仅统计本网关承诺范围内**的处理耗时；不包含外部知识/模型服务时间 | 准备固定规则与固定问题集；在约定并发下进行压测/复测，输出 P99 |
| NFR-P02 | **智能问答路径**：未命中规则，需要走外部知识/模型能力 | **P99 < 100ms** | 统计范围为：从北向接口被调用起，至把问题交给外部 RAG 服务之前；该区间**包含**未命中规则后的**向量语义检索**（含本网关触发的侧车 Embedding 与向量库检索等网关侧环节），以体现**网关整体对外**耗时；**明确不包含**外部 RAG/知识模型服务自身处理时间 | 使用“未命中规则”的问题集（含需走向量检索的样本）；对外部 RAG 采用**可控替身**（固定延迟或记录拆分）后测量本网关承诺部分 P99 |

## 5. 非功能需求（产品级，不含技术实现）

| 编号 | 类别 | 需求描述 | 验收标准（产品） |
|---|---|---|---|
| NFR-G01 | 可用性 | 在依赖的外部服务异常时，用户应收到明确失败提示，而不是长时间无响应 | 故障演练下，超时前有可理解提示 |
| NFR-G02 | 安全（管理侧） | 仅系统管理员可变更规则库；普通用户不可篡改 | 未授权访问被拒绝 |
| NFR-G03 | 数据一致性（管理侧） | 规则变更不应导致“偶发旧规则/新规则混用”不可解释 | 变更后用同一问题重复验证结果稳定一致（在“生效时效”到达后） |
